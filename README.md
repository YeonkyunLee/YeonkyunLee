# 안녕하세요 👋 (Yeonkyun Lee)

임베디드에서 시작해 **DSP 알고리즘**(PCR/qPCR 신호처리)을 개발하고, 지금은 **머신러닝**과
**로봇 상태추정**으로 영역을 넓히고 있습니다. 관심의 중심은 *신호가 발생하는 곳에서
실시간으로 도는, 정확하고 안전한 시스템* — DSP · ML · 임베디드 · 진단(규제) 도메인이
만나는 지점입니다.

- 🔬 신호처리 → ML: 고전 DSP와 학습 기반 방법을 **같은 지표로 정직하게** 비교
- 🤖 추정이론 → 로봇: 칼만부터 그래프 SLAM까지 **밑바닥부터** 구현
- ⚙️ 임베디드 감각: 엣지 실시간·풋프린트를 항상 함께 검증
- ✅ 규제 도메인: "깨끗해 보이지만 틀린" 결과를 잡는 안전장치에 관심

> 관통하는 원칙 하나: **좋아 보이는 것과 실제로 옳은 것은 다르며, 그 간극은 지표
> 하나로는 안 보인다.** 그 간극을 드러내는 실험을 설계하는 걸 좋아합니다.

## 🧪 signal-ml-lab — 고전 DSP vs ML로 ECG 디노이징·진단
[**github.com/YeonkyunLee/signal-ml-lab**](https://github.com/YeonkyunLee/signal-ml-lab)

DSP 개발자가 ML로 넘어가며, 생체신호를 소재로 두 접근을 정면 비교한 엔드투엔드 실증.
공개 데이터(PhysioNet)·합성만 사용.

- ML이 인분포에서 고전을 **+10~15 dB** 압도 → 그러나 OOD/실데이터에서 **조용히 붕괴**
- **딥앙상블 안전게이트**(AUROC 0.90~0.96)로 그 실패를 사전 감지
- 실 MIT-BIH+실노이즈로 **sim-to-real 격차** 실증, 합성=데이터효율 prior
- 부정맥 분류(환자 독립), **디노이징이 진단을 해치는** 시스템 함정 → **공동학습**으로 해결
- 전체 파이프라인 **1285 beats/s(430× 실시간)** — 웨어러블 온디바이스 가능
- 📓 7편 블로그 시리즈로 전 과정 기록

## 🤖 sensor-fusion-lab — 칼만부터 그래프 SLAM까지
[**github.com/YeonkyunLee/sensor-fusion-lab**](https://github.com/YeonkyunLee/sensor-fusion-lab)

DSP의 최적 필터를 상태공간으로 확장해 로봇 상태추정 전 과정을 밑바닥부터 구현.

- 선형 KF · 위치+IMU 융합(단일센서 압도, outage 관성주행)
- 비선형 **EKF/UKF**(CTRV), **IMU 바이어스** 온라인 추정(관측성 시각화)
- **EKF-SLAM** 자기위치+지도(오도메트리 17× 개선) — 정직한 디버깅 여정 기록
- **루프 클로저**(부분) → **그래프 SLAM**(pose-graph, 루프클로저 1개로 전체 궤적 정렬, 5×)
- 📓 EKF-SLAM 디버깅 여정 포함 블로그 시리즈
